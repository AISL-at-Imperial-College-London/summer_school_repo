Student tasks

  1. Manual search: move m and b until the line looks reasonable. What happens to the point in parameter space?
  2. Read the contours: change only the slope, then only the intercept. Which direction in parameter space does each action follow?
  3. Find the valley: try to reach an MSE below — without revealing the optimum.
  4. Optimisation: reset the parameters, then take gradient steps. Explain why the path tends to cut across contours.
Key connection: one point (m,b) in parameter space corresponds to one complete straight line in data space.
Start by moving either slider.

For teaching: ask students to predict the direction of the next gradient step before pressing the button.

The optimum is computed from the ordinary least-squares solution; gradient descent is shown only as an iterative route to the same minimum.
