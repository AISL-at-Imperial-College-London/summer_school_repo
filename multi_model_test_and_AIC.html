A more flexible model should not be preferred merely because it fits the training data better. It should be preferred only when its extra flexibility improves prediction of unseen data.

7Information criteria: does the extra parameter help?

A more flexible model will normally fit the training data at least as well. The Akaike Information Criterion penalises this additional complexity:

AIC = 2k − 2ℓmax
Here, k is the number of fitted parameters and ℓmax is the maximised training-data log-likelihood. Lower AIC is better.

Why is Gamma preferred when its log-likelihood gain exceeds 1?

Gamma has exactly one more fitted parameter than Exponential. Therefore,

AICG < AICE   exactly when   ℓG − ℓE > 1.

The extra parameter costs 2 AIC units, so it must improve the maximised log-likelihood by more than 1.

Small-sample correction
AICc = AIC + 2k(k + 1) n − k − 1

With only a small number of training observations, AICc is more appropriate because it penalises additional parameters more strongly.

How to use this alongside the test data: AIC and AICc assess fit versus complexity using the training data. Since this exercise already reserves unseen observations, the held-out log-likelihood remains the primary predictive comparison.
